# B站弹幕宠物挂机

## 功能

- 进入配置的直播间，**每 10 分钟**自动执行：
  1. 直播间粉丝勋章签到
  2. 发送直播弹幕「签到」
  3. 发送直播弹幕「修炼」，按 12 秒 × 最多 2 轮校验经验增量
  4. 读取宠物经验，如命中突破条件则发送直播弹幕「突破」并复查是否真的升级

- 直播间关播后自动切换到其他开播直播间，继续执行上述流程
- 蛋宠面板现在主要用于读取经验、判断突破和复查结果，不再承担签到 / 修炼 / 突破的主触发通道

- Cookie 登录校验失败时，若已配置 `QQ_MAIL_USER` / `QQ_MAIL_PASS`，会自动发送失效通知邮件

- 省额度版默认：每次 GitHub Actions 运行最多 25 分钟，每天触发 2 次（北京时间 08:00 / 20:00）



## 配置项（GitHub Secrets）

| Secret 名称 | 说明 |
|---|---|
| `BILIBILI_COOKIE` | B站登录 Cookie（完整字符串） |
| `BILIBILI_UID` | B站 UID（可选，辅助心跳） |
| `HANGUP_ROOM_ID` | 首选直播间房间号（若未开播则自动换房） |
| `QQ_MAIL_USER` | QQ邮箱地址（Cookie 失效通知，可选） |
| `QQ_MAIL_PASS` | QQ邮箱授权码（可选） |

## 运行方式

- **定时**：每天北京时间 08:00 / 20:00 自动触发
- **手动**：GitHub Actions → Run workflow，可自定义：
  - `cycle_minutes`：每轮间隔（默认 10 分钟）
  - `max_runtime_minutes`：最大运行时长（默认 25 分钟；手动复核时可临时调高）


## 费用估算（私有仓 2000 分钟/月）

| 方案 | 触发频率 | 每次时长 | 估算月消耗 |
|---|---|---|---|
| 当前省额度版 | 每天 2 次 | 25 分钟 | **约 1500 分钟/月** |

> 这是按 `2 × 25 × 30 ≈ 1500` 做的保守估算，留出约 500 分钟给手动调试、失败重跑和 `keepalive`。比之前“每 6 小时触发一次、单次 350 分钟”的长跑方案省很多，也更接近 GitHub Free 私有仓 2000 分钟额度上限。


## 日志说明

- 每轮结果直接打印到 Actions 日志
- 格式：`[北京时间] 操作描述`，`✅` 表示成功，`⚠️` 表示轻微异常，`❌` 表示严重错误
