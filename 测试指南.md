# 测试指南

本文档只覆盖当前仍在使用或仍有排查价值的能力：
- **B站弹幕宠物挂机修炼**
- **宠物签到（本地辅助）**
- **边界AI每日签到**

> 单独的 **B站每日登录经验签到** 已移除，不再作为常规测试项。

## 一、本地测试

### 前提条件

1. 已安装 Node.js 16+
2. 已执行 `npm install`
3. 已从 `.env.example` 复制出 `.env`
4. `.env` 中至少填好了 `BILIBILI_COOKIE` / `BILIBILI_UID`

### 1）测试挂机修炼主链路

```bash
npm run hangup
```

重点看日志里是否出现：
- 成功命中可用直播间
- 发出修炼指令
- 第 1 次 12 秒复查后若仍像 `+14` 基础收益或出现 `+14` 倍数，会继续自动做第 2 次 12 秒复查
- 两轮复查后的累计经验增量达到有效修炼档位，且不是 `+14` 的倍数
- 满经验时触发突破链路

### 2）测试宠物签到（本地辅助）

```bash
npm run pet
```

重点看：
- 是否成功进入可用弹幕宠物房间
- 是否成功发送「签到」弹幕
- 是否没有出现未登录 / 频控 / 房间不可用等错误

### 3）测试边界AI签到

```bash
npm run yyai:signin
```

重点看：
- 返回是否明确成功
- 或者明确显示“今日已签到”
- 如果失败，是否打印出 token 相关错误

### 4）测试默认入口

```bash
npm start
```

当前默认只跑：
- **B站弹幕宠物挂机修炼**

如果你想本地同时跑挂机和宠物签到，可在运行前设置：

```bash
TASK=all node src/index.js
```

> Windows PowerShell 可写成：

```powershell
$env:TASK='all'; node src/index.js
```

## 二、GitHub Actions 测试

### 1）手动触发挂机修炼

1. 进入仓库 **Actions**
2. 选择 **B站弹幕宠物挂机修炼**
3. 点击 **Run workflow**
4. 等待执行完成

验收重点：
- 工作流成功结束
- 日志中能看到候选房间扫描或优先房间命中
- 返回摘要里能看到触发顺序 / 实际触发方案 / 经验增量结果

### 2）手动触发边界AI签到

1. 进入仓库 **Actions**
2. 选择 **边界AI每日签到**
3. 点击 **Run workflow**
4. 查看结果

验收重点：
- 成功签到
- 或明确显示“今日已签到”
- 失败时能看到 token 失效或接口返回原因

## 三、常见问题

### 问题1：登录验证失败

常见表现：
- `Cookie验证失败`
- `账号未登录`
- `登录验证失败，退出执行`

处理：
1. 重新获取 B站 Cookie
2. 更新 `BILIBILI_COOKIE`
3. 确认 Cookie 中包含 `SESSDATA`、`bili_jct`、`DedeUserID`

### 问题2：挂机跑了但经验没到有效修炼档位

常见表现：
- 只看到基础在线收益
- 没有达到有效修炼增量

处理方向：
1. 先看命中的是否是优先房间
2. 再看实际触发的是面板还是直播弹幕
3. 对照返回摘要里的经验增量判断是否真生效

### 问题3：边界AI签到失败

常见表现：
- `token` / `access-token` 失效
- 返回 unauthorized / invalid / expire

处理：
1. 打开 `https://yyai8.com/signIn`
2. 重新抓取 `token` / `access-token` / `uid`
3. 更新到 GitHub Secrets 或本地 `.env`

## 四、最小验收清单

### B站挂机修炼

- [ ] 能成功进入可用直播间
- [ ] 能发出修炼指令
- [ ] 能看到经验增量校验结果
- [ ] 满经验时能触发突破逻辑

### 边界AI签到

- [ ] 能成功签到或识别“今日已签到”
- [ ] token 失效时能给出明确报错

### 配置层

- [ ] `BILIBILI_COOKIE` 已配置
- [ ] `BILIBILI_UID` 已配置
- [ ] `YYAI_TOKEN` / `YYAI_ACCESS_TOKEN` / `YYAI_UID` 已配置（若启用边界AI）
- [ ] 邮件通知配置正确（若启用提醒）

## 五、当前原则

- 不再把“表面成功”当成功
- B站侧优先看真实经验增量
- 不再为单独 B站每日登录经验工作流做排查
