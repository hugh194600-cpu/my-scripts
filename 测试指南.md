# 测试指南

## 当前架构

- **运行方式**：GitHub Actions 长驻运行（单次约 350 分钟），每6小时触发一次
- **核心逻辑**：进入直播间 → 每 10 分钟循环（弹幕签到 + 弹幕修炼 + 弹幕突破触发，经验仍靠面板复查）→ 关播自动换房


- **入口文件**：`src/index.js`（精简版，约 400 行）

---

## 每轮循环流程

```
[进入直播间]
    ↓
[检查直播间是否在播] → 关播 → 换房 → 重新进入
    ↓
发心跳（保活）
    ↓ （等 3s）
直播粉丝勋章签到（API）
    ↓ （等 3s）
尝试读取蛋宠面板（拿经验基线）
    ↓
发送弹幕「签到」
    ↓ （等 3s）
发送弹幕「修炼」
    ↓ （12s 后复查经验；若仍像 +14 基础收益，再补 1 轮 12s 复查）
读取宠物经验
    ↓
命中突破条件？
  是 → 发送弹幕「突破」→ 15s 复查 → 确认升级
  否 → 等待到下一轮（10 分钟间隔）

```


---

## 手动触发测试

GitHub Actions → `B站弹幕宠物挂机` → Run workflow

可选参数：
- `cycle_minutes`：每轮间隔，填 `2` 可快速验证 2～3 轮后退出
- `max_runtime_minutes`：最大运行时长，填 `5` 可快速跑一轮后退出

| 前缀 | 含义 |
|---|---|
| `✅` | 操作成功 |
| `⚠️` | 轻微异常（会继续） |
| `❌` | 严重错误（可能退出） |
| `🎉` | 命中突破条件 |

## 环境变量参考

| 变量名 | Secret / Env | 说明 |
|---|---|---|
| `BILIBILI_COOKIE` | Secret | B站完整 Cookie |
| `BILIBILI_UID` | Secret | B站 UID（可选） |
| `HANGUP_ROOM_ID` | Secret | 首选直播间号 |
| `QQ_MAIL_USER` | Secret | QQ邮箱（可选，Cookie 失效时发送通知） |
| `QQ_MAIL_PASS` | Secret | QQ邮箱授权码（可选） |

| `CYCLE_MINUTES` | Workflow input | 每轮间隔，默认 10 |
| `MAX_RUNTIME_MINUTES` | Workflow input | 最大运行时长，默认 350 |
| `DANMU_GAP_MS` | Env（可选）| 面板操作间隔毫秒（沿用旧变量名），默认 3000 |

## 常见问题

**Q: 宠物经验读取失败？**  
A: 检查 `HANGUP_ROOM_ID` 对应直播间是否开启了弹幕宠物互动游戏；尝试在浏览器里直接访问直播间确认。




**Q: 直播间换了但还是失败？**  
A: 新换的直播间可能没有弹幕宠物，可在 `HANGUP_ROOM_ID` 里配置一个确认开启了弹幕宠物的直播间。
